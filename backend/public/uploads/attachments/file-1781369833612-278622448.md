## 1. Melhorias na Interface de Conversas e Tickets

### 1.1. Retirar a palavra "trial" em conversas
* **Descrição:** A flag indicando que a conta está em modo *trial* não deve aparecer nas janelas de conversa do usuário/atendente.
* **Arquivos Prováveis:** * `frontend/src/components/ChatWindow.tsx`
  * `frontend/src/components/TicketDetail.tsx` ou similar.
* **Ação:** Localizar o componente visual de header e criar uma condicional para ocultar a tag "trial" quando a visualização não for estritamente do Admin financeiro, ou removê-la definitivamente da interface de chat.

### 1.2. Layout de Políticas de SLA e Tags
* **Descrição:** Colocar políticas de SLA ao lado de Tags no mesmo layout.
* **Arquivos Prováveis:** * `frontend/src/components/TicketDetail.tsx` (ou o painel de detalhes na lateral direita).
* **Ação:** Refatorar o contêiner flex/grid que envolve estes componentes para que `Tags` e `SLA` ocupem a mesma linha (ex: `<div className="flex flex-row justify-between gap-4">...</div>`).

### 1.3. Tradução do Painel de Detalhes (Inglês -> Português)
* **Descrição:** Colocar em português os termos em inglês que aparecem na lista de tickets, no painel direito de detalhes.
* **Arquivos Prováveis:**
  * Componentes do painel lateral (como `TicketDetail.tsx` ou os componentes de propriedades do ticket).
* **Ação:** Substituir strings hardcoded no frontend (ex: *Status*, *Assignee*, *Created at*) pelas chaves de tradução correspondentes em pt-BR.

## 2. Correções no Admin e Superadmin

### 2.1. Funcionalidade de Lixeira para Empresas
* **Descrição:** No painel Admin não é possível mandar empresa (Tenant) para a lixeira.
* **Arquivos Prováveis:** * `admin/frontend/src/components/SuperAdmin/TenantsTab.tsx`
  * Backend: `backend/src/services/tenantService.ts` e `backend/src/routes/admin.ts`.
* **Ação:** * **Backend:** Garantir que o endpoint `DELETE /tenants/:id` implemente o *soft delete* (já existe um script SQL `08-trash-system.sql`, confirmar se a coluna `deleted_at` está sendo alimentada no tenant).
  * **Frontend (Admin):** Adicionar/corrigir a chamada de API no botão de exclusão e atualizar o estado da tabela após a exclusão.

### 2.2. Correção de Data de Trial da Empresa
* **Descrição:** Não é possível corrigir a data de trial da empresa via painel Admin.
* **Arquivos Prováveis:**
  * Backend: Endpoints de atualização de Tenant / Subscrição (`subscriptionService.ts` / `tenantService.ts`).
  * Frontend Admin: Modal de edição de empresa (`TenantModal.tsx` ou `TenantDetailsModal.tsx`).
* **Ação:** Habilitar um *date picker* para o campo `trialEndDate` no frontend do Admin e assegurar que o payload preserve e grave essa atualização no banco de dados.

### 2.3. Fluxo de Anexos em Conversas (Superadmin)
* **Descrição:** Ao anexar um arquivo na conversa, o sistema seleciona o arquivo e depois mostra uma tela inesperada/confusa.
* **Arquivos Prováveis:**
  * Frontend: Lógica de upload no `ChatWindow.tsx` ou componente de Upload.
* **Ação:** Revisar o fluxo e remover a abertura indevida de modais posteriores à seleção. Apenas deve exibir a pré-visualização (se for imagem, ex: `ImageViewerModal.tsx`) ou enviar diretamente o anexo, deixando o comportamento claro.

### 2.4. Erro Genérico em Preenchimento (Bug Visual)
* **Descrição:** Tela de erro/bloqueio ao preencher determinados campos (conforme Image 6 e 8 do relatorio).
* **Ação:** Capturar os *logs* de requisição para este formulário específico. Provavelmente, uma falha de validação de campo (ex: CPF/CNPJ via `cpfValidation.ts`) ou formato de data que está quebrando o parse no frontend sem o devido tratamento de erro (try/catch na interface com `<Toast />`).

## 3. Fluxo de Renovação e Assinaturas (Billing)

### 3.1. Tratamento de Assinatura Vencida
* **Descrição:** 1. O sistema não dá a possibilidade de renovar por mais 7 dias no primeiro vencimento.
  2. No segundo vencimento, deve direcionar para uma tela de "Condição Especial".
* **Arquivos Prováveis:**
  * Backend: Middleware de checagem (`backend/src/middleware/validateMw.ts` ou serviço de assinatura) e `subscriptionService.ts`.
  * Frontend: O roteador principal e a tela `SpecialOffer.tsx` (que já existe na pasta `frontend/src/`).
* **Ação:**
  * Criar lógica que contabiliza as extensões de trial (ex: flag `trial_extended_count`).
  * Se `trial_extended_count == 0` ao vencer, o modal/banner de expiração mostra o botão **"Renovar por +7 dias"**.
  * Se `trial_extended_count > 0`, interceptar o login/acesso e forçar o redirecionamento para a rota `/special-offer`.

## 4. Onboarding de Novas Contas

* **Descrição:** Colocar opção na fase de popular banco de dados para criar 1000 contatos, 5000 mensagens e 5000 tickets.
* **Arquivos Prováveis:**
  * `frontend/src/Onboarding.tsx`
  * `backend/src/services/demoDataService.ts`
  * `backend/src/routes/onboarding.ts`
* **Ação:**
  * **Frontend:** Adicionar um *radio button* ou *checkbox* na tela de Onboarding para a escolha do tamanho da base (Pequena vs. Grande).
  * **Backend:** Alterar o `demoDataService.ts` para aceitar um parâmetro de volume. Implementar o *insert* em **lote (bulk insert)** utilizando transações ou processamento em fila (`queue.ts`) para evitar *timeout* na conexão caso a criação de 11.000+ registros leve mais tempo que o limite do endpoint HTTP.
